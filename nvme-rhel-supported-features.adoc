= Obtenga información sobre la compatibilidad y las funciones de ONTAP para hosts RHEL
:hardbreaks:
:toclevels: 1
:allow-uri-read: 
:toclevels: 1
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ./media/


[role="lead"]
Las características compatibles con la configuración del host con NVMe over Fabrics (NVMe-oF) varían según su versión de ONTAP y RHEL.

[cols="50,25,25"]
|===
| Característica | Versión del host RHEL | Versión ONTAP 


| Se admite la autenticación segura en banda a través de NVMe/TCP entre un host RHEL y un controlador ONTAP  a| 
9.3 o posterior
| 9.12.1 o posterior 


| NVMe/TCP proporciona espacios de nombres utilizando el protocolo nativo `nvme-cli` paquete  a| 
8.2 o posterior
| 9.10.1 o posterior 


| NVMe/TCP es una función empresarial totalmente compatible  a| 
9.0 o posterior
| 9.10.1 o posterior 


| Se admite el tráfico NVMe y SCSI en el mismo host mediante NVMe multipath para espacios de nombres NVMe-oF y dm-multipath para LUN SCSI  a| 
8.2 o posterior
| 9.4 o posterior 
|===
ONTAP admite las siguientes funciones de host SAN independientemente de la versión de ONTAP que se ejecute en la configuración de su sistema.

[cols="75,25"]
|===
| Característica | Versión del host RHEL 


| La multirruta nativa de NVMe está habilitada de forma predeterminada  a| 
10.0 o posterior



| El arranque SAN se habilita mediante el protocolo NVMe/FC  a| 
9.4 o posterior



| El `nvme-cli` El paquete incluye scripts de conexión automática que eliminan la necesidad de scripts de terceros.  a| 
8.2 o posterior



| La regla nativa de udev en `nvme-cli` El paquete proporciona equilibrio de carga round-robin para rutas múltiples NVMe  a| 
8.2 o posterior



| La regla nativa de udev en `nvme-cli` El paquete proporciona equilibrio de carga de profundidad de cola para rutas múltiples NVMe  a| 
9.6 o posterior

|===

NOTE: Para obtener detalles sobre las configuraciones admitidas, consulte lalink:https://mysupport.netapp.com/matrix/["Herramienta de matriz de interoperabilidad"^] .
